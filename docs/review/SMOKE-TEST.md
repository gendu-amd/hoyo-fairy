> 本文档分两部分：

## 0. 前提

- 待测文件：仓库根目录 **`biliHoyoFairy.user.js`**（唯一、自包含、已内联全部模块）。
- 当前版本 **`0.0.6`**（有行为变更，已升版）；油猴会按 `@updateURL` 自动更新，本地测试请**手动覆盖**确保是最新产物。
- 测试期间**只开这一个 B 站屏蔽脚本**，关掉其它同类脚本，避免互相争抢 DOM。

## 1. 安装（覆盖现有版本）

**办法 A（推荐）复制粘贴覆盖：**
1. Cursor 里 `Cmd/Ctrl+P` 打开**仓库根**的 `biliHoyoFairy.user.js`（不是 `src/` 里的）。
2. `Cmd/Ctrl+A` 全选 → 复制。
3. 浏览器 → 油猴图标 → 管理面板 → 点开已装的「biliHoyoFairy」→「编辑器」标签 → 代码区全选删除 → 粘贴 → `Cmd/Ctrl+S` 保存。

**办法 B：** 资源管理器右键根目录 `biliHoyoFairy.user.js` → Download → 把下载的 `.user.js` 拖进浏览器 → 油猴「重新安装/覆盖」。

**确认装的是新包**：油猴编辑器里搜一下，能看到 `// src/constants.ts`、`// src/match/engine.ts` 这种**模块边界注释**，且头部 `@version` 为 `0.0.6`。

装好后打开 `https://www.bilibili.com/`（**无后缀首页**），F12 开控制台，按下面跑。

## 2. 冒烟清单（勾选 + 预期）

### A. 启动 & 控制台（最关键，先看这个）
- [ ] 控制台出现一行 `[biliHoyoFairy] v0.0.6 已启动 | 页面:首页 | 拦截:开`（版本号即当前 `@version`）
- [ ] **启动时控制台没有任何红色报错**（尤其 `ReferenceError: xxx is not defined` —— 这是打包漏接线的典型症状）
- [ ] 右下角出现 🛡 角标

> ⚠ 若这一组就报错，先把报错原文贴给我，后面不用测了。

### B. 拦截层 → 计数 → 角标（seam: net→stats→badge）
- [ ] 首页推荐流正常加载，没有大面积空白/破版
- [ ] 角标显示「🛡 已拦截 N（共 M）」，且**刷新/下滑会看到 N 在涨**（说明网络层在删项、stats 在记账、角标在更新）
- [ ] 切到 搜索页 / 播放页（右侧推荐）/ 排行榜 / 热门 / 动态，各页推荐流都正常、该拦的拦

### C. 打开面板（seam: toast→hooks→panel）
- [ ] **点 🛡 角标能打开设置面板**（这步验证「角标→打开面板」的注入桥）
- [ ] 面板里基础/黑名单/进阶/评论/白名单/工具各分组都能展开，内容正常

### D. 加规则即时生效（seam: rules→events→dom）
- [ ] 黑名单里随便加一个**当前首页可见 UP 的名字**或一个高频关键词 → **不刷新页面，命中的卡片立刻消失/标记**
- [ ] 删掉该规则 → 被隐藏的卡片（DOM 层那种）恢复
> 这步验证「改规则 → 发事件 → 重建匹配器 + 重扫」整条链路，是断 dom↔rules 环后的核心回归点。

### E. 右键菜单 & 拉黑（menu / blacklist）
- [ ] 视频卡上右键 → 弹出自定义菜单（屏蔽UP / 拉黑 / 加白名单 / 隐藏这一张 / 打开设置）
- [ ] 「🚫 屏蔽UP」→ 该卡消失、toast 提示
- [ ] 「🙈 隐藏这一张」→ 仅这张消失
- [ ]（**登录态下，谨慎**）随便挑一个不在意的 UP 点「⛔ 拉黑」→ toast 提示已拉黑/已在黑名单；到 [官方黑名单页](https://account.bilibili.com/account/blacklist) 能看到（测完可手动移除）

### F. 悬停拉黑浮层（menu 浮层）
- [ ] 面板「基础」里打开「悬停显示拉黑按钮」→ 鼠标移到卡片上出现「⛔ 拉黑」浮层；移开消失

### G. 评论区过滤（comments + shadow 钩子）
- [ ] 打开任意视频播放页，面板「评论」开启「启用评论区过滤」+ 勾一个条件（如「纯表情评论」或加个评论关键词）
- [ ] 命中的评论被**折叠成一行灰条**（点击能展开）；关掉过滤后评论恢复
> 验证 attachShadow 钩子 + 评论模块联动。

### H. 订阅 / 预置 / 导入导出（panel + subscriptions + config）
- [ ] 「工具」→ 点一个预置规则库按钮（如库洛系）→ 规则灌入对应黑名单、即时生效
- [ ] 「工具」→ 导出配置 → 能弹出/复制 JSON；导入一段 → 规则合并
- [ ]（可选）添加一条订阅 URL → 能拉取并合并。注意：自 0.0.6 起 `@connect` 已去掉 `*`，仅声明 B 站 + 常见 CDN 域名；自定义域名的订阅/名单 URL 首次访问时 Tampermonkey 会弹**一次性授权确认**（选「总是允许」即可），属正常。

### I. 名单批量处理（panel 批量 + blacklist）
- [ ] 「工具」→ 名单批量处理 → 粘贴几个 UID/UP 名 → 「仅屏蔽（本地）」→ 提示新增 N 条、即时生效

### J. 屏蔽记录 & 计数（panel + stats）

- [ ] hover 记录条能看到完整标题/BV/UID

### K. 审查模式（dom 标记分支）
- [ ] 面板开「审查模式」→ 被判定拦截的卡**不隐藏、而是描边+原因+就地放行按钮**

## 3.5 v0.0.6（Phase 2）新增/变更项 — 重点重测

> 这些是 0.0.6 引入的**行为变更**，与 0.0.5 不同，需单独覆盖一遍。

- [ ] **单条拉黑二次确认（F）**：在 feed 上右键某卡 →「⛔拉黑UP…」/ 悬停拉黑按钮 → 应先弹确认框；取消则不发生任何拉黑；确认后才执行。
- [ ] **批量拉黑可停止（G）**：面板「名单批量处理」粘贴多个 UID →「⛔拉黑」→ 执行中出现「⏹停止」且「拉黑」按钮禁用；点停止 → 等当前这条收尾后停止，状态显示「已停止（已处理 x/总数）」，未处理 + 失败的 UID 回填到输入框。
- [ ] **当前页批量进度（G）**：「拉黑当前页所有已屏蔽的 UP」执行中，按钮文字实时变「拉黑中 x/n…」（或风控暂停提示），完成后恢复。
- [ ] **日限提示（G）**：名单粘贴 >200 个 UID 时，确认框出现「账号黑名单总量上限/建议分批」提示。
- [ ] **批量一次性重扫（A）**：批量拉黑/仅屏蔽完成后页面只整体重扫一次（不应每条都闪一次），结果与逐条一致。
- [ ] **feed 逐项容错（J）**：（难手测，主要由单测 + 调试日志保证）正常浏览首页/搜索时拦截照常工作，不因偶发异形数据整页失效。
- [ ] 以下项由代码常量/单测保证、无需手测：缓存容量上限、`uidNames` 软上限、对比度 WCAG AA、ReDoS 正则上限/启发式。
- [ ] **联合投稿（K）**：开启「联合投稿连带拉黑」后拉黑一个多人投稿 → 提示「已拉黑 x/总数 位作者」数字正确（不再恒为 0 或错乱）。
- [ ] **导入剥离订阅（D）**：导入一份**手动加了 `subscriptions` 字段**的配置 JSON → 导入后「规则订阅」列表不应被对方的订阅污染（subscriptions 被丢弃）。
- [ ] **@connect 收窄（C）**：从一个**非 B 站/非内置 CDN** 域名「从 URL 载入」名单或加订阅 → Tampermonkey 弹一次性授权确认（选「总是允许」后正常拉取）。
- [ ] **正则上限（E）**：在规则里粘贴一条超长 `/.../`（>1000 字符）→ 页面不卡死，该行被忽略，其它规则照常生效。

### v0.0.6 体验项（Phase 4）

- [ ] **样式化确认弹窗**：上述所有确认（右键/悬停拉黑、名单批量、当前页批量、删订阅、恢复默认、清空列表、预置开精确过滤）弹出的是**与面板同风格的弹窗**，不再是浏览器原生灰框。
- [ ] **弹窗键盘/遮罩**：弹窗出现时按 `Esc`=取消、`Enter`=确认；点弹窗外的灰色遮罩=取消；危险操作（拉黑/删除/恢复默认/清空）默认焦点在「取消」。
- [ ] **toast 配色**：拉黑成功=绿色左条、未登录/账号侧失败=橙色、网络错误/触发风控=红色左条；普通提示仍为原样。
- [ ] **拉黑撤销（toast）**：单条拉黑成功（已登录）后，弹出的 toast 带「撤销」按钮；点它 → 提示「已撤销拉黑」，刷新后该 UP 恢复推荐。
- [ ] **拉黑撤销（面板）**：打开面板「屏蔽记录」，刚拉黑的（标「黑」）条目右侧有「↩撤销」；点 → 确认弹窗 → 撤销成功，该 UP 移出账号黑名单。
- [ ] **面板记录拉黑确认**：「屏蔽记录」里对非账号拉黑条目点「⛔拉黑」→ 先弹确认弹窗（与右键一致）。
- [ ] **暗色模式**：系统切到深色后，面板/弹窗/toast/右键菜单整体为深底浅字，无残留刺眼白块；切回浅色恢复。
- [ ] **键盘**：打开面板后按 `Esc` 关闭；`Tab` 可在面板内移动且有焦点环；有确认弹窗时 `Esc` 先关弹窗（不会连面板一起关）。
