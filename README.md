# hoyoFairy · 抗击黑潮

> 凯撒命你兵分五路抗击黑潮，此乃其一。

[![Install](https://img.shields.io/badge/Tampermonkey-一键安装-fb7299)](https://raw.githubusercontent.com/gendu-amd/hoyo-fairy/main/hoyo-fairy.user.js)
[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

🛡 一个 Tampermonkey 用户脚本，专治 B 站推荐流里的**黑流量、引战视频、商业广告**，并屏蔽你**不想看的 UP 主**：在 B 站请求推荐数据时就按你的规则把命中项删掉（渲染前拦截），漏网的再在 DOM 层兜底隐藏，并能一键将 UP 主同步拉黑到你的账号黑名单。

覆盖 **首页 / 热门 / 排行榜 / 搜索页 / 播放页推荐**，兼容 **Edge / Chrome / Firefox**（装 Tampermonkey 即可，无需单独的浏览器扩展）。

## 为什么需要它

B 站自带的「关键词屏蔽 / 不感兴趣」只在展示环节隐藏标题命中的卡片，不回流推荐模型，刷新后照样复现。本脚本不依赖平台：**拦截优先**——在 B 站自身请求推荐流 JSON 时就把命中项删掉，页面只渲染保留项（无遮罩、无留白、无闪烁）；**DOM 兜底**——首屏 SSR、搜索页、需联网取数的进阶维度等拦不到的，再在页面里安全隐藏。而真正让内容「从推荐流消失」，靠的是**一键拉黑**（写入账号黑名单，刷新后 B 站不再推荐）。

> 当前为 **pre-release（v0.0.1）**，优先打磨「在各页面可靠识别并干净过滤 + 真·拉黑」这一核心。部分能力（规则订阅、营销号自动识别、跨标签页同步）尚未实现，待核心稳定后逐个加回。

## 功能（已实现）

**两层过滤架构**

- **拦截层（主）**：hook `fetch` / `XHR`，在渲染前从首页推荐 / 排行榜 / 热门 / 播放页相关推荐的 JSON 里删除命中项
- **DOM 兜底层（薄）**：隐藏拦截层覆盖不到的卡片（首屏 SSR、搜索页、进阶联网维度），与拦截层共用同一套规则
- **审查模式**（面板可开）：命中卡不删不隐，而是描边标记 + 就地「放行」按钮，便于核对、防误伤

**精准拦截**

- 关键词：一次命中 标题/UP名/分区，`/.../` 包裹按正则（默认大小写不敏感）
- UP 主名（精确）、UP 主 UID（精确）、BV 号
- 时长过滤（最短/最长秒数）、播放量阈值（低于 X 万）

**卡片类型过滤（开关即用）**

- 广告/推广卡（含 `is-rcmd` 内联推广）、搜索框热搜词

**🛰 进阶过滤（API 维度，默认关，开启才发请求）**

- **视频标签黑名单**：标题区看不到的标签也能拦（`/.../` 支持正则）——治"换个标题继续推"
- **双重标签·治引战**：`原神+鸣潮` 形式，两组标签**同时命中**才拦，专治对立引战内容
- **UP 简介关键词**：按 UP 个人简介拦营销号
- **充电专属视频**
- 内置 **缓存 + 限速队列**（`view` / `tag` / `card` 三接口），防风控；命中后卡片稍后隐藏

**核心能力**

- ⭐ **白名单优先**：关键词/UP/UID 白名单命中的视频永不隐藏，避免误伤
- **右键一键操作**：卡片任意处右键 → 屏蔽 UP / 拉黑 UP / 加白名单 / 屏蔽此视频 / 隐藏这一张
- **一键拉黑（真·移除）**：调用官方接口 `/x/relation/modify (act=5)` 同步账号黑名单（未登录则仅本地屏蔽）
- **预置关键词库**：营销号UP名 / 标题党 / 软传销 / MBTI / 梗视频 / 含日语标题 / 寄生社蛆，一键加入
- **屏蔽记录**：面板内查看本次会话拦了哪些（分类统计 + 明细），每条带来源徽章（拦截层 NET / DOM 层）与「放行」按钮
- **导入 / 导出**：把规则与过滤开关导出为 JSON 分享，导入时与本地配置合并去重（剔除统计/缓存/个人会话偏好）
- **实时生效**：`MutationObserver` 监听无限滚动新卡片；规则即改即生效
- **数据本地**：规则仅存浏览器本地（`GM_setValue`），不外传

## 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/) 扩展。
2. **Edge 用户**：在 `edge://extensions/` 启用「开发人员模式」（否则 MV3 下用户脚本可能不生效）。
3. 点击安装脚本，Tampermonkey 会弹出安装页，确认安装：
   - **GitHub**：[安装脚本](https://raw.githubusercontent.com/gendu-amd/hoyo-fairy/main/hoyo-fairy.user.js)
   - **Greasy Fork（国内/打不开 GitHub 推荐）**：[安装脚本](https://greasyfork.org/zh-CN/scripts/582873)
4. 打开 [bilibili.com](https://www.bilibili.com/)，右下角出现 🛡 角标即成功。

> **pre-release 提示**：当前 `@version` 暂保持 `0.0.1` 不变，Tampermonkey 不会自动升级。想测最新代码请手动覆盖安装仓库里的 `hoyo-fairy.user.js`。核心稳定后会改回正常版本号 + 自动更新。

## 使用

1. 点右下角 🛡 角标打开面板。
2. **最快上手**：点「预置关键词库」按钮（如 `+ 寄生社蛆`），规则立即生效。
3. **手动加规则**：输入框打字 → 「添加」或回车 → 变成可删除的标签，即时生效。关键词支持 `/正则/`。
4. **右键操作（推荐）**：在卡片任意处右键，选「屏蔽 UP / 拉黑 UP / 加白名单 / 屏蔽此视频 / 隐藏这一张」。
5. **真·移除**：对反复出现的恶心 UP，右键选「拉黑 UP」（或在「屏蔽记录」里拉黑），刷新后 B 站不再推荐。
6. **保护喜欢的**：把 UP/关键词加进 ⭐ 白名单，永不被误删。
7. **排查**：开「调试模式」看控制台逐卡日志，或在面板「屏蔽记录」查看实际拦了什么。

## 注意事项

- 不要同时启用多个 B 站屏蔽类脚本，否则会争抢首页 DOM 导致行为错乱。
- 请使用无后缀首页 `https://www.bilibili.com/`（带 `index.html` 的可能不生效）。
- Chromium 内核建议版本 ≥ 105（`:has()` 等能力依赖）。
- 一键拉黑依赖登录态（读取 `bili_jct` Cookie）；未登录时仅本地拦截。
- B 站前端改版可能使选择器失效，届时更新脚本即可。

## 参与开发（欢迎贡献）

欢迎提 Issue 反馈 bug / 建议，或提 Pull Request 改代码。完整流程见 [CONTRIBUTING.md](CONTRIBUTING.md)，简版：

```
Fork → 新建分支 → 改 hoyo-fairy.user.js → 本地装脚本验证 → 提 PR
```

- **报告问题**：用 [Issue 模板](.github/ISSUE_TEMPLATE/) 提交，附上页面类型、复现步骤、控制台 `[hoyoFairy]` 日志（开「调试模式」更易定位）。
- **本地测试**：把仓库里的 `hoyo-fairy.user.js` 拖进 Tampermonkey 安装/覆盖，刷新 B 站验证。
- ⚠️ **发布即自动更新**：`@updateURL` 指向 `main` 分支，任何合入 `main` 的改动都会被所有用户自动拉取。因此 PR 务必经 Review + 实测；每次对外发布请记得给 `@version` +1，否则用户不会升级。
