# 更新日志

本项目遵循语义化版本（pre-release 阶段为 `0.0.x`）。`@version` 单一来源在 `src/meta.js`。

## [0.0.6] - 2026-06-24

Phase 2：安全与正确性强化（有行为变更，需重新冒烟）。

### 安全
- **收窄网络权限**：`@connect` 去掉 `*`，仅声明 `api.bilibili.com` 及常见 CDN 域名。自定义域名的订阅/名单 URL 首次访问会由 Tampermonkey 弹一次性授权确认。
- **导入剥离不可移植键**：配置导入侧与导出侧对称剔除 `NON_PORTABLE`（尤其 `subscriptions`），防止他人分享的「规则文件」借导入悄悄塞入会自动联网拉取的订阅 URL。
- **正则 ReDoS 防护**：订阅/导入来源的 `/正则/` 模式体超过 `MAX_REGEX_LEN`（1000）时忽略，避免灾难性回溯卡死页面。

### 正确性
- **联合投稿回调修正**：`blacklistUp` 联合投稿路径改用结果对象 `{added,already,total,failed}`，修复旧版误按 `(n,total)` 取参导致文案与回调恒错。
- **批量拉黑一次性重扫**：批量本地屏蔽不再逐条存盘 + 全页重扫，改为收尾统一一次，消除大批量时的卡顿。
- **feed 逐项容错**：拦截层逐项 `try/catch`，单条畸形数据不再放弃整条响应的其余过滤。

### 安全 / 体验
- **单条拉黑二次确认**：右键菜单与悬停浮层的拉黑执行前需确认（账号写操作不可一键撤销）。
- **批量拉黑可停止**：名单批量新增「停止」按钮，停止时把未处理 + 失败的 UID 回填输入框便于续传；`doBlacklistMany` 返回可取消的控制器。
- **批量进度**：当前页批量在按钮上实时显示进度 / 风控暂停状态。
- **日限提示**：名单数量较多（>200）时提示账号黑名单总量上限与单日大批量易触发风控，建议分批。

### 体验 / 可达性
- **样式化确认弹窗**：所有确认对话框（拉黑、删订阅、恢复默认、清空列表、开启精确过滤等）从原生 `confirm()` 升级为与面板同风格的弹窗，支持 `Esc` 取消 / `Enter` 确认、点击遮罩取消，危险操作默认聚焦「取消」降低误确认，`role=dialog`/`aria-modal` 提升可达性。
- **toast 分级配色**：动作结果提示按结果上色——风控/网络错误=红、未登录/账号侧失败=橙、拉黑成功/已在黑名单=绿，一眼可辨（中性提示保持原样）。
- **拉黑「撤销」**：把不可逆的账号拉黑变为可恢复——单条拉黑成功后弹出带「撤销」按钮的 toast（8 秒），面板「屏蔽记录」中已拉黑的条目常驻「↩撤销」按钮；撤销走 `relation/modify act=6`，账号侧移出黑名单 + 本地恢复，刷新后该 UP 恢复推荐。面板记录里的「⛔拉黑」也补齐了二次确认。
- **暗色模式**：自有 UI 全表面跟随系统 `prefers-color-scheme` 暗色适配（面板/弹窗/toast/右键菜单/输入/按钮/chip/Tab/滚动条），品牌色与语义色保留。
- **键盘可达性**：面板 `Esc` 关闭、打开时焦点移入面板、关闭时归还焦点；关闭 ✕ 支持 `Enter`/`Space`；为按钮/Tab/链接等补 `:focus-visible` 焦点环。
- **对比度**：提示/统计/空态/分组提示等说明文字提到 WCAG AA（≥4.5:1）。

### 性能 / 内存
- **API 缓存容量上限**：`view`/`tag`/`card` 缓存改用带上限的 `capMapSet`（800/1200/800），超限按插入顺序淘汰最旧键，避免长会话内存无界增长（淘汰项下次重取，结果不变）。
- **`uidNames` 软上限**：持久化的 UP 名缓存上限 5000；已存在键照常更新，仅新增键达上限时跳过，避免存档 blob 无界膨胀。
- **观察器（评估后保持现状）**：`MutationObserver` 仍按批全量 `scanAll`，但单卡昂贵判定已被 `PROCESSED` 短路、每批仅一次原生 `querySelectorAll`（开销极小）。增量化重写收益微小却有 shadow/skeleton 卡片覆盖回退风险，故暂不改，待真实 profiling 证据再议。
